# 《天下布武》原創視覺 Art Bible

> - 狀態：M6-V1 基線（2026-07-14）
> - 適用：戰略地圖、PixiJS 場景元件、React HUD、圖示與後續原創素材
> - 上位契約：`plan/04-map-and-movement.md`、`plan/11-ui-screens.md`、`plan/12-ui-components.md`
> - 參考研究紀錄：`docs/visual/references.md`

## 1. 設計命題

本作的視覺命題是「**低飽和戰國沙盤 × 現代戰略資訊層**」。玩家第一眼應看見地形與水系，第二眼讀到道路、城池與領域，第三眼才是軍隊、命令與警告。HUD 退到畫面邊緣，詳細數值只在 hover、選取、分析圖層或情境面板中展開。

歷史感來自和紙、墨、木、漆、印章與少量金箔的原創材質語彙，不來自仿製既有商業遊戲的版面、圖示、城模、肖像、貼圖或配色。本文件定義的是可實作與可驗收的視覺規則；任何新語彙須先回寫本文件或 `plan/12-ui-components.md`，再進入 production。

### 1.1 閱讀優先序

1. **地貌骨架**：海岸、山系、主要水系與平原開口。
2. **戰略網路**：主幹道、橋樑／渡口、城池、領域與要衝。
3. **即時態勢**：軍隊、方向、命令、補給與戰鬥警告。
4. **精確資訊**：名稱、兵力、耐久、士氣、ETA 與操作提示。

若兩項在同一縮放層級爭奪注意力，順位較低者必須降對比、延後顯示或移入情境 UI。常駐 HUD 不得比選取中的城池或軍隊更醒目。

## 2. 單一真相與實作邊界

- UI 色彩、字體、間距、陰影與動效的實作值以 `src/ui/styles/tokens.ts` 為準。
- 地圖層序、鏡頭門檻、領地透明度與命中半徑以 `plan/04-map-and-movement.md` 為準。
- HUD 尺寸、1280×720 縮放策略與畫面互動以 `plan/11-ui-screens.md` 為準。
- 圖示 master grid、共用元件與素材 manifest 契約以 `plan/12-ui-components.md` 為準。
- 本文件新增的地圖色票是 **M6-V5／M6-V6 的美術目標**；實作前須加入相應 token 或 `mapViewConfig`，禁止在繪製程式中散落硬編碼色碼。
- 地圖資產只表達 view，不得承載規則真相；兵力、耐久、道路狀態等仍來自 selector／view contract。

### 3.3 對比與色弱規則

- 文字對比依 `plan/12-ui-components.md` 的 gate；金色文字一律用 `accentGoldText`。
- 敵對：朱紅 **加尖角旗型／交叉兵器**；友軍：靛藍 **加圓角雙環**；中立：灰 **加空心菱形**。
- 成功／失敗：除綠／紅外，分別加入勾形／驚嘆三角，並保留可讀標籤。
- 道路等級以線寬、雙線／單線、節線共同編碼；海路另有波節，不只換顏色。
- 選取態以金色雙輪廓、輕微外擴與固定焦點標記共同呈現，不以提高飽和度代替。

## 4. 材質、光照與深度

### 4.1 材質語彙

| 材質 | 使用處                         | 限制                                                        |
| ---- | ------------------------------ | ----------------------------------------------------------- |
| 和紙 | HUD 面板、地圖低頻底紋         | UI 5% 等效雜訊；地圖紋理頻率更低，overview 不可形成雪花噪點 |
| 墨   | 文字、輪廓、山脊、道路 casing  | 邊緣可有 1–2% 不規則，但命中與幾何仍由乾淨向量控制          |
| 木   | 次級頁籤、分隔條、城下町暗示   | 不使用寫實木紋照片；以原創幾何與低對比筆觸表達              |
| 漆   | 主要控制列、選取節點的深色基座 | 小面積使用，避免整張 HUD 變成高反差黑框                     |
| 金箔 | 選取、稀有、權威層級           | 不超過單畫面可視 UI 面積的 3%；禁止大面積漸層金             |

所有 bitmap 素材須能追溯 manifest。可程式生成的紙紋、山紋、波紋與道路刷痕，優先以自製 SVG／Canvas 程序生成，避免無法驗證來源的外部貼圖。

### 4.2 光照規則

- 採固定的「左上柔光、右下短陰」；所有城池、旗幟、badge 與浮起面板方向一致。
- 地圖是策略沙盤，不做動態日夜循環；季節只以低幅度色溫與少量地景差異呈現，不改變資訊色。
- 地形 relief 使用兩階明暗，不做寫實 PBR。山脊陰影 alpha 建議 12–20%，不得遮住道路與領地界線。
- 發光只保留於選取、威風、火災與重要警告；常態 icon、城池、道路不使用 bloom。
- 陰影不越級：地圖物件的投影小於 HUD 浮層，HUD 浮層小於 modal。

## 5. 三段 LOD

LOD 使用 `scale < 0.5`、`0.5 ≤ scale < 1.0`、`scale ≥ 1.0` 三段，切換保留 10% hysteresis。LOD 的目的不是把重要層整層關掉，而是替換 detail variant。

| 視覺域     | overview／far                      | operational／mid           | close／near                        |
| ---------- | ---------------------------------- | -------------------------- | ---------------------------------- |
| 地形／水系 | 海岸、主要山系、主要河川；大塊明暗 | 加森林群、次水系、關隘地形 | 加聚落、農田、林緣與局部紋理       |
| 領地       | 勢力大面與深色邊界                 | 加郡界／選取郡             | 加局部控制、分析圖層細節           |
| 道路       | 主幹道與海路；名稱隱藏             | 加次道、橋樑、方向與目的地 | 加小路、道路名、每日刻度與命中提示 |
| 城池       | 本城、勢力輪廓、被圍／選取         | 加支城、平城／山城、城名   | 加耐久、警戒、城下地景與互動提示   |
| 軍隊       | 全部部隊、敵我、主方向；聚合標籤   | 加兵力級距、命令、補給警告 | 加士氣、精確兵力、ETA 與疊放展開   |
| 標籤       | 國名／勢力名、選取物               | 城名、目的地、重要警告     | 道路名、郡名、精確狀態             |

共同規則：

- overview 仍須看見主幹道、城池與軍隊，禁止退回「只有彩色多邊形」。
- 任何縮放層級，選取物與 critical 警告都可突破一般隱藏規則，但不得遮蔽操作目標。
- 文字不是縮小後硬塞；低於可讀尺寸時改為短標籤、badge、聚合或隱藏。
- 同一物件跨 LOD 的錨點與主輪廓保持穩定，避免縮放時視覺跳位。

## 6. 五類地圖物件語言

### 6.1 地形與水系

**原創母題**：分層紙雕等高面＋短墨脊。山地以 2–4 層錯位剪影構成，不畫寫實山峰；森林以群塊冠幅與林緣節奏構成，不鋪滿單株樹；平原保持大片安靜負空間。

- 海岸：深海底＋陸側 2–4px 暖灰內陰影；海岸線不得用高亮白邊。
- 河川：上游細、下游寬；橋樑處水線連續但被橋面壓過。
- 山地：overview 只留山系走向；close 才增加短脊與稀疏陰影。
- 森林：群塊外輪廓比內部紋理重要；領地 tint 疊加後仍可辨識。
- 次要地景 alpha 低於道路與節點，不得造成「地形很漂亮但路看不見」。

### 6.2 領地與分析覆蓋

**原創母題**：染紙而非油漆。勢力色像滲入地圖紙張的薄染，地形明暗仍穿透；邊界採局部墨深，不使用霓虹發光線。

- 一般領地 alpha 0.45；overview 0.65；勢力分析模式 0.85。
- 無主郡用中性灰，未知情報用斜向稀疏網點；兩者不可只靠灰階深淺。
- 分析主題同時最多一個為主高亮，其餘回到基礎層。
- 圖例必須同步顯示紋樣與文字，不只展示色塊。

### 6.3 道路、橋樑與海路

**原創母題**：墨線護邊的沙盤交通脈絡。每條道路由 casing＋內線構成，waypoints 呈現地形順應，不用端點直線穿山越海。

| 類型   | 輪廓              | 內線             | 次級通道                      |
| ------ | ----------------- | ---------------- | ----------------------------- |
| 主幹道 | 3–5px 深墨 casing | 寬暖金實線       | overview 保留；節點用雙短刻痕 |
| 次道   | 2–3px casing      | 較窄棕金實線     | operational 起顯示            |
| 小路   | 1–2px casing      | 短節線           | close 才顯示                  |
| 海路   | 深海藍外線        | 長節線＋週期波節 | 全程與陸路形狀不同            |
| 阻斷   | 原線保留          | 斷口＋叉形封印   | 加原因 tooltip，不只轉紅      |

橋樑是水面上方的短矩形橋面與兩端橋頭，不使用單一圓點代替。可互動路點的可見 glyph 可小，但透明 hit area 至少 32 CSS px。

### 6.4 城池、郡與聚落

**原創母題**：俯視沙盤中的紙雕城郭，不是寫實天守模型。

- 本城：外郭五角／不規則城郭＋內部二階主郭；支城：較簡單的四角郭。兩者在 overview 只看輪廓也能分辨。
- 平城：水平展開、矮郭、護城河短弧；山城：垂直堆疊、岩基三角與窄郭。類型不只靠文字。
- 歸屬：旗面／郭內小面積勢力色；輪廓仍用 `ink900`，避免不同勢力變成完全不同資產。
- 耐久：close 以環形缺口／段數顯示；overview 只有被圍或瀕危才顯示狀態環。
- 受攻：固定裂口徽記＋低頻朱紅脈衝；警戒：金色小烽火；選取：金色雙環。
- 城下聚落使用低對比屋頂群與田畦，只作空間語意；不可比城池本體更清楚。
- 郡節點採空心圓印＋中心點，與城郭輪廓明確分族。

現有 `CastleNode` 的勢力色填面、墨色描邊與耐久環可作契約基礎；M6-V7 應替換幾何細節，不改 selector 的 tier／durability 語意。

### 6.5 軍隊、命令與戰鬥狀態

**原創母題**：以旗幟為主體的行軍棋子。旗桿提供方向基準，旗面表達歸屬，尾形與徽記表達關係／狀態；不使用人物頭像或寫實兵團縮圖。

- 常規隊：燕尾旗；軍團／大將隊：橫腰帶；敵對接觸：旗尾尖角；敗走：旗面下垂 20°。
- 行軍方向：旗座前方的短箭頭＋路徑前 1 段；靜止時箭頭隱藏。
- 兵力：overview 用 3 級旗幅／聚合數；operational 顯示縮寫；close 顯示精確值。
- 士氣：沿用 3 點語法，但低士氣加破裂外框，高士氣加上揚旗結，不只換顏色。
- 補給：正常不顯示；偏低為米袋缺口；危急為空袋＋驚嘆三角。
- 狀態 badge 固定在棋子右上，最多同時顯示一個最高優先狀態；其餘進 tooltip／選取面板。
- 8 支以上重疊時用確定性扇形偏移與數量 badge；被選取軍隊永遠置頂，標籤避讓不得改變世界座標。
- 動效：行軍 2–3px 低幅 bob；接戰短促震動；`prefers-reduced-motion` 下改為靜態徽記。

現有 `ArmyChip` 的旗桿、燕尾旗、軍團腰帶與三點士氣可保留為資料映射基礎；M6-V8 必須增加方向、補給、狀態與色弱次級通道。

## 7. 城池與軍隊視角

- 投影固定為 **俯視 35° 的假等角紙雕**：可看見頂面與短前緣，但不建立真 3D 相機。
- 城池錨點在郭心底部；軍隊錨點在旗桿落地處；道路錨點在中心線。選取環以錨點為中心，不跟投影陰影偏移。
- 所有物件光源皆由左上；投影向右下 2–6px，隨 LOD 變化但方向不變。
- 城池的視覺高度永遠低於同位置軍隊旗桿，避免軍隊被城模吞沒。
- 部隊通過城池時使用 z-sort／固定偏移保持旗面可見，不改遊戲位置。

## 8. 六類 UI 圖示系統

### 8.1 共通網格

- master：24×24，四周 2px safe area；主要筆畫落在 4–20 範圍。
- `stroke: currentColor`、基準線寬 1.75、round cap／join；必要的地圖／軍事輪廓可有小面積實心，但不可混成另一套粗細。
- 16px 版刪除內部細節並做半像素光學校正；24px 為標準；32px 版可增加一級資訊，禁止直接等比縮放全部細節。
- 名詞優先單一強剪影；動詞採「名詞主體＋一個方向／變化記號」。一枚 icon 不超過三個語意元件。
- 可見 glyph 與 hit target 分離；控制面至少 32×32 CSS px，無文字 icon 必帶 `ariaLabel` 與 Tooltip。

### 8.2 六族定義

| 圖示族     | 視角／骨架              | 代表 glyph                                     | 原創辨識規則                                                       |
| ---------- | ----------------------- | ---------------------------------------------- | ------------------------------------------------------------------ |
| 導航／時間 | 正交、水平／垂直中軸    | close、play、pause、ff2、ff5、chevron、arrow   | 箭頭尖角一致；倍速以重複箭體＋末端刻線，不用文字 `2x` 當主圖示     |
| 資源／內政 | 正面或輕俯視、圓潤容器  | coin、rice、people、scroll、plus、minus        | 資源主體配一個計量特徵；避免寫實貨幣紋章                           |
| 軍事／外交 | 斜 45° 動勢、較尖外輪廓 | flag、army、sword、siege、crown、handshake     | 敵意由尖角／交叉表達，合作由相扣形表達；禁止沿用史實家紋剪影       |
| 地圖／圖層 | 俯視、堆疊平面          | castle、road、bridge、terrain、layer、pin      | 與場景物件共享語意但降低細節；圖層用錯位紙片，不用通用漢堡選單代替 |
| 狀態／警告 | 徽章容器＋單一核心符號  | supply、morale、warning、blocked、eye、eye-off | warning 固定三角；blocked 固定斜封條；不同嚴重度仍保留同一輪廓     |
| 系統／資料 | 正交、規整 4／8 向節奏  | gear、search、book、save／load（後續）         | 最低裝飾量；系統操作不可借用軍事圖像造成誤判                       |

### 8.3 狀態變體

- default：`ink500`；hover：`washi200` 底；focus：既有雙層 focus ring。
- selected／toggled：金色 20% 實心底＋深金 glyph＋外框，不只換 glyph 顏色。
- disabled：保留完整輪廓、降低對比、禁止用刪除線偽裝；Tooltip 說明原因。
- critical：三角或裂口附加記號＋朱紅；不可把整枚原 icon 換成純紅圓點。
- unread：右上 6–8px 實心朱印點，主 glyph 不位移。

## 9. HUD safe area 與資訊密度

### 9.1 基準版面

1920×1080、`uiScale=1`：上列 48px、左列 72px、右上通知 320px、右下 MiniMap 224px、選取時底部情境面板 168px。1280×720 自動縮放為 0.667：上列 32px、左列 48px、情境面板 112px，地圖可視區至少 1160×512。

### 9.2 安全區規則

- **戰場焦點區**：未選取物時，主要交戰與選取目標應落在扣除上列、左列與右側通知／MiniMap 後的最大中央區；鏡頭 focus command 必須考慮 HUD occlusion。
- 通知預設最多顯示規格數量，critical 可常駐但不得向中央無限延伸；多餘項收進報告中心。
- 未選取任何物時底部面板整塊隱藏。選取後鏡頭不自動把目標留在面板後方。
- 目標選擇模式的頂部橫條不得蓋住路徑終點；必要時相機將路徑 bounds 置於剩餘可視區。
- tooltip、label 與 badge 要避開常駐 HUD 邊界 8px；不可被裁切在 canvas／DOM 交界。
- overview 不以右側長列表取代地圖；操作細節進情境面板，地圖保持視覺主體。

### 9.3 密度預算

- 常駐 HUD 只顯示日期、核心資源、速度、七個主入口、通知與 MiniMap。
- 單一地圖物件在未選取時最多同時有：主輪廓、名稱、1 個最高優先 badge、1 條關係線。
- 同一格 64×64 CSS px 內最多一個完整文字標籤；其餘用避讓、聚合或選取後展開。
- 最低可見文字 12 CSS px；小於此值時改用 glyph，不縮成不可讀字。

## 10. 字體、標籤與數字

- 全域 `Noto Serif TC`，一般內文 400；標題的 700 只在字型資產確實提供時使用。
- 地圖標籤以 `ink900`＋`washi100` 3px halo，保持穿越領地色與地形時可讀。
- 數字使用 tabular numerals；兵力縮寫規則在同一 LOD 必須一致，不混用 `3,000`、`3k`、`三千`。
- 城名／道路名沿線不旋轉超過 ±30°；更陡時保持水平並以 leader line 指向。
- 國名／勢力名是 overview 的低對比大字，不得蓋過本城與軍隊。

## 11. 動效與回饋

- 一般 hover 150ms、狀態變化 300ms、鏡頭聚焦依現有 400–600ms 契約；重要威風演出最多 800ms。
- 地圖常態動效只允許行軍、選取、戰鬥／攻城與必要警告。地形、道路與普通城池保持安靜。
- 呼吸效果週期至少 1.2s，alpha 變化不超過 20%；禁止多處同步閃爍。
- reduced motion：取消 bob、震動、粒子與連續脈衝，保留最終狀態輪廓、徽記與文字。
- 操作回饋先改輪廓／狀態，音效與粒子只是增強，不能成為唯一回饋。

## 12. 參考與禁抄清單

### 12.1 可以萃取

- 特定縮放下「保留哪些資訊」的層級原則。
- 行軍、攻城、補給、領地等資訊同時出現時的閱讀問題。
- HUD 與地圖之間的視覺權重、遮蔽與操作衝突。
- 玩家能否快速辨識城池、軍隊、道路、方向與警告的可用性觀察。

### 12.2 禁止直接沿用

- KOEI TECMO 或其他遊戲的 screenshot、裁圖、貼圖、shader 輸出、3D 城模、角色肖像、家紋與音效。
- 原作面板的比例、位置組合、角飾、框線、按鈕排法、色彩數值與動效節奏。
- 原作 icon 的輪廓、節點配置、描邊比例，包含描摹、翻轉、改色、加邊框等衍生方式。
- 以生成式工具要求「完全像某遊戲 UI／城池／地圖」；提示詞只可描述本文件的原創材質、資訊需求與幾何語彙。
- 未核對授權便收進 repo、Figma、參考資料夾或訓練集的外部圖像。

### 12.3 原創轉譯流程

1. 在 `references.md` 登錄 URL、存取日與來源類型，不下載原圖進 repo。
2. 只寫抽象觀察，例如「overview 仍保留主道路」，不記錄可複製的像素尺寸與色碼。
3. 對照本文件選擇原創母題、次級辨識通道與 LOD 行為。
4. 以灰階 wireframe／幾何草稿先驗證辨識，再套本作 tokens 與材質。
5. 新資產進 manifest，記錄作者／工具、來源、授權、衍生狀態與內容 hash。
6. 以固定 visual fixture 做三段截圖、色弱、對比與遮蔽檢查。
